
| Название эксперимента                                        | Описание                                                                                                                                                                                                                                                                     | Затрагиваемый метод                                                                                           |
| -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| Первое сканирование пустой`src_d`                      | Убедиться, что при пустой входной директории не создаётся образцов, дерево сохраняется пустым.                                                                                                       | `scan_filesystem`, `_process_initial_tree`, `save_to_db`                                                                  |
| Первое сканирование с тестовыми образцами   | Поместить готовую структуру`DNA/TEST/sample1/batch1/pod5/…`. Должен создаться один образец с корректными `sample_id`, `batches`, `source_data` и `history`.                                               | `scan_filesystem`, `_process_initial_tree`, `_create_sample`, `_get_sample_file_size`                                   |
| Повторное сканирование без изменений            | Запустить второй цикл без модификации файлов. Никаких новых образцов, нет изменений в существующих.                                                                                              | `_compare_and_process_tree`, `_mark_sample_changed`                                                                         |
| Добавление нового батча                                     | В существующий образец добавить папку нового батча (например,`batch2/pod5/…`). Должен обновиться состав батчей, размер образца, появиться запись в истории.   | `_compare_and_process_tree`, `_mark_sample_changed` (change_msg "New batches")                                              |
| Изменение размера файлов в батче                     | Добавить или заменить файлы в папке`pod5`, чтобы изменился общий размер. Размер образца должен обновиться.                                                                                     | `_compare_and_process_tree` (блок `size_changed`), `_mark_sample_changed`                                             |
| Удаление целого батча                                         | Удалить папку одного из батчей. Батч должен быть удалён из состава образца, размер пересчитан.                                                                                                         | `_compare_and_process_tree` (блок `removed_batches`), `_mark_sample_changed`                                          |
| Удаление директории образца                             | Удалить всю папку`sample1`. Образец должен быть помечен как удалённый (`source_removed=True`), размер обнулён.                                                                                                   | `_compare_and_process_tree` (блок `removed_folders` на глубине sample), `_mark_sample_changed(deleted=True)` |
| Стабильность новых папок (флаг`_is_stable`)          | Создать новую папку батча с «будущей» датой модификации (моложе`min_stable_time`). Папка не должна быть добавлена в дерево до истечения времени стабильности. | `_compare_and_process_tree` (проверка `_is_stable` для `new_batches` и `new_folders`)                       |
| Загрузка ранее сохранённого дерева из БД      | Перезапустить вотчдог. Он должен загрузить сохранённое дерево и не создавать дубликаты образцов.                                                                                                  | `load_preindexed_from_db`, `_load_tree`, первая ветка `scan_filesystem` (old_tree is not None)                 |
| Одновременное добавление и удаление батчей | В одном цикле добавить новый батч и удалить старый. Оба изменения должны быть зафиксированы корректно.                                                                                         | `_compare_and_process_tree` (одновременные `new_batches` и `removed_batches`)                               |
