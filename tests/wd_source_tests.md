| Название эксперимента | Описание | Действия | Пройдено | Затрагиваемый метод |
|----------------------|----------|----------|----------|----------------------|
| Первое сканирование с тестовыми образцами | При первом запуске и отсутствии сохранённого дерева вотчдог должен рекурсивно обойти `src_d`, создать объекты `Sample` для всех обнаруженных образцов и сохранить их в БД. | 1. Очистить коллекции `samples` и `file_trees`. 2. Поместить готовую структуру папок образца в `src_d` (например, `DNA/TEST/770130661501/...`). 3. Запустить приложение. 4. Дождаться первого цикла `source_wd`. 5. Проверить: в БД появился документ с `sample_id=770130661501`, `source_d`, `batches`, `history` с записью "Sample created", дерево в `file_trees` сохранено. | Да | `scan_filesystem`, `_process_initial_tree`, `_create_sample`, `_save_tree`, `save_to_db` |
| Повторное сканирование без изменений | При отсутствии изменений в файловой системе повторный цикл не должен создавать новые образцы или обновлять существующие. | 1. Дождаться следующего цикла после первого сканирования (без модификации файлов). 2. Проверить логи: "changed=False" на всех уровнях. 3. Убедиться, что `upsert_many` не вызывается (сообщение "Нет документов для вставки"). | Да | `_compare_and_process_tree`, `save_to_db` |
| Добавление нового батча | При добавлении новой стабильной папки на уровень батча существующего образца система должна зафиксировать новый батч и обновить образец (размер, история). | 1. Создать новую папку внутри образца (например, `770130661501/new_batch/pod5/...`). 2. При необходимости выждать время стабилизации или изменить mtime (`touch -t`). 3. Дождаться цикла `source_wd`. 4. Проверить: в логах "New batches: [new_batch]", образец в БД обновлён, размер пересчитан, запись в истории. | Да | `_compare_and_process_tree`, `_mark_sample_changed` |
| Удаление батча | При удалении папки существующего батча образец должен обновиться: батч удалён из списка, размер пересчитан, запись в истории. | 1. Удалить папку одного из батчей внутри образца. 2. Дождаться цикла `source_wd`. 3. Проверить: в логах "Removed batches: [...]", образец обновлён, размер уменьшен. | Да | `_compare_and_process_tree`, `_mark_sample_changed` |
| Удаление образца | При удалении всей папки образца из `src_d` образец должен быть помечен как удалённый (`source_removed=True`), размер обнулён, запись в истории. Поля `work_d`, `res_d` не должны изменяться. | 1. Удалить папку образца (например, `770130661501`). 2. Дождаться цикла `source_wd`. 3. Проверить: в логах "Sample directory removed", документ в БД: `source_removed=True`, `source_d_size_GB=0.0`, `work_d` и `res_d` сохранили прежние абсолютные пути. | Да | `_compare_and_process_tree`, `_mark_sample_changed`, `save_to_db` |
| Восстановление ранее удалённого образца | Если папка удалённого образца возвращается на место, система должна обнаружить её, восстановить образец (`source_removed=False`), пересчитать размер и добавить запись в историю. | 1. После успешного удаления (предыдущий тест) вернуть папку образца. 2. Дождаться цикла `source_wd`. 3. Проверить: в логах "Restored previously deleted sample", документ в БД: `source_removed=False`, размер > 0, запись "Sample directory restored". | Да | `_create_sample`, `_mark_sample_changed` |
| Стабильность новых папок (проверка `_is_stable`) | Недавно созданные папки (возраст меньше `min_stable_time`) должны игнорироваться; после истечения времени стабилизации (или принудительного старения через `touch -t`) — обрабатываться. | 1. Создать новую папку батча (например, `test_batch`) без изменения mtime. 2. Дождаться ближайшего цикла. 3. Убедиться, что папка не добавлена (лог: `stable=False`, "removing from tree"). 4. Состарить папку (`touch -t 202001010000 test_batch`) или дождаться `min_stable_time`. 5. Дождаться следующего цикла. 6. Проверить, что папка теперь добавлена (лог: `stable=True`, батч добавлен). | Да | `_is_stable`, `_compare_and_process_tree` |
| Загрузка сохранённого дерева при перезапуске | При перезапуске приложения вотчдог должен загрузить ранее сохранённое дерево из БД и корректно сравнить его с текущим состоянием ФС, не создавая дубликатов образцов. | 1. Выполнить полный цикл сканирования и сохранения (любой из предыдущих тестов). 2. Перезапустить приложение. 3. Дождаться первого цикла после перезапуска. 4. Проверить: в логах "Loaded stored tree: present", образцы не создаются заново (если ФС не изменилась). | Да | `load_preindexed_from_db`, `_load_tree`, `scan_filesystem` |
| Автоматическое определение размера с вложенными папками | Метод `_get_sample_file_size` должен корректно суммировать размеры файлов, даже если батч содержит подпапки с файлами. | 1. Внутри батча создать подпапку (например, `pod5/`), в которой лежат файлы. 2. Дождаться цикла. 3. Убедиться, что размер образца вычислен правильно, без ошибок типа `Unexpected type in batch size data` (лог должен быть чистым от таких предупреждений). | Да | `_get_sample_file_size` |
